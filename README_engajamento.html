README do projeto

People Analytics Dashboard
Glassdoor, Benchmark e LinkedIn

Aplicação Shiny para acompanhamento de reputação empregadora, satisfação interna percebida, comparação com concorrentes e indicadores sintéticos de audiência no LinkedIn. O projeto foi preparado para rodar de forma demonstrativa, sem depender de arquivos externos de dados.

R Shiny bslib Plotly DT Dados sintéticos

Visão geral

Este dashboard consolida análises de marca empregadora e presença digital em quatro blocos: avaliações no estilo Glassdoor, benchmark competitivo, visão geral de LinkedIn e demografia de visitantes/seguidores. A interface usa tema escuro, cards de KPI, gráficos interativos e tabelas filtráveis.

Importante: os dados são gerados por funções internas do próprio código. O app não precisa de planilhas, CSVs, banco de dados, credenciais ou arquivos auxiliares para iniciar.

Estrutura recomendada da pasta

Para publicar ou executar como app Shiny de pasta, deixe apenas o código R e este README:

projeto-glassdoor-linkedin/
├── app.R
└── README.html

O arquivo enviado pode ser mantido como app_glassdoor.R, mas para rodar uma pasta Shiny diretamente é recomendado renomeá-lo para app.R.

Funcionalidades principais

PáginaO que mostra
Análise Glassdoor KPIs de NPS, satisfação geral, liderança, CEO, recomendação, avaliações positivas/negativas e total de avaliações. Inclui projeção da satisfação, NPS por dimensão, evolução temporal, comparação entre funcionários atuais e ex-funcionários, NPS por área e tabela de dimensões.
Benchmark Compara a empresa com concorrentes sintéticos por classificação geral, carreira, remuneração, cultura, liderança, qualidade de vida e recomendação. Inclui radar, barras comparativas e tabela.
Visão Geral LinkedIn Acompanha visualizações, visitantes únicos, seguidores, crescimento mensal, origem do tráfego, acesso por dispositivo e composição por tamanho de empresa.
Demografia LinkedIn Analisa visitantes e seguidores por experiência, função, tamanho de empresa e evolução mensal, com botões para alternar entre públicos.

Dados sintéticos

As bases são criadas em memória durante a inicialização do app. As principais funções são:

  • gerar_dados_glassdoor(): avaliações, cargos, status, notas por dimensão e sentimento.
  • gerar_dados_concorrentes(): tabela de benchmark com empresas anonimizadas.
  • gerar_dados_linkedin_visao_geral(): seguidores, novos seguidores e engajamentos mensais.
  • gerar_dados_linkedin_visitantes_serie(): visualizações, visitantes únicos e dispositivo.
  • gerar_demografia_linkedin(): funções, localidades, tamanhos de empresa, experiência e setores.
  • load_all_data(): orquestra a geração e prepara os objetos usados no dashboard.

Filtros disponíveis

  • Status: todos, atual ou ex-funcionário.
  • Sentimento: todos, positivo, neutro ou negativo.
  • Ano: permite restringir as avaliações por ano.
  • Demografia LinkedIn: filtros próprios de ano e mês nas visualizações demográficas.

Instalação dos pacotes

Execute uma vez no R antes de abrir o app:

install.packages(c(
  "shiny", "bslib", "ggplot2", "dplyr", "lubridate", "readr", "readxl",
  "tidyr", "stringr", "plotly", "shinyWidgets", "scales", "forecast",
  "DT", "tm", "RColorBrewer", "purrr", "zoo", "forcats", "janitor"
))

Como executar

  1. Crie uma pasta para o projeto.
  2. Coloque o código R na pasta e renomeie para app.R.
  3. Coloque este arquivo como README.html na mesma pasta.
  4. Abra o R nessa pasta e execute:
shiny::runApp(".")

Também é possível abrir o arquivo no RStudio e clicar em Run App.

Customização rápida

  • Altere o nome exibido no cabeçalho procurando por EMPRESA 1.
  • Ajuste o volume de avaliações em gerar_dados_glassdoor(n = 500).
  • Modifique cargos, áreas e dimensões dentro das funções de geração sintética.
  • Troque metas e cores dentro das funções auxiliares de KPI, se necessário.

Troubleshooting

ProblemaComo resolver
Erro “there is no package called ...” Instale o pacote indicado ou rode novamente o bloco de instalação.
Caracteres acentuados aparecem incorretos Salve o arquivo R como UTF-8. Em alguns ambientes Windows, a localidade pode exigir ajuste manual.
O app não abre ao rodar a pasta Confirme que o arquivo principal se chama app.R e que ele termina com shinyApp(ui = ui, server = server).
Gráficos vazios Verifique se os filtros selecionados não restringiram demais a amostra sintética.

Privacidade e uso

Este projeto é demonstrativo. Os nomes de empresas, cargos, avaliações, seguidores, visitantes e indicadores são fictícios ou anonimizados. Nenhum dado real precisa ser usado para executar o app.
